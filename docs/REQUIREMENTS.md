## I. 国際化（i18n / l10n / CJK）
- **UI文字列の外部化・多言語（Fluent等）** [MVP後] [seam:小] `t!("key")` 間接層を最初から、ハードコード禁止。
- **CJK禁則処理・行分割（kinsoku、UAX#14、約物詰め）** [差別化★（日本語品質）] [seam:Yes] 行分割ポリシーを差し替え可能な境界に。
- **縦書き（tategaki）/縦中横** [差別化/MVP後・要注意] [seam:Yes★] レイアウト座標系がwriting-modeを一級市民に。横書き前提でハードコードすると縦書きは事実上不可能。
- **RTL/双方向（bidi）** [MVP後] [seam:Yes] 縦書きと共通の書字方向抽象に内包。
- **ロケール別 数値/日付/通貨（ICU4X）** [MVP後] [seam:No] 表計算時に本番。
- **フォントフォールバック/カラー絵文字** [MVP] [seam:Yes] 豆腐□回避は必須。フォント章と統合。

## J. フォント管理

- **シェーピング（合字/結合/CJK、rustybuzz/HarfBuzz）** [MVP] [seam:Yes] テキストレイアウト中核に内包。

## K. テキストレイアウト中核（横断・差別化の本丸）
§I・§J は1つの中核に収斂: **シェーピング + フォント解決/フォールバック + CJK禁則 + writing-mode**。
gpuiのtext_systemがグリフ整形/フォールバックを担うが、**禁則・縦書き・bidiのポリシー層は別途必要になり得る**（DESIGN §8.2参照）。
